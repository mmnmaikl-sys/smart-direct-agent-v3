## 1. Внутренняя механика автостратегий (Абрамова, урок 1 + Идалин)

### Как алгоритм принимает решения

Автостратегии -- алгоритмические модели, которые в реальном времени управляют ставками на основе:

1. **Исторических данных**: клики, конверсии, поведение пользователей за последние 7-28 дней
2. **Конкурентного окружения**: текущие ставки конкурентов в аукционе
3. **Бюджетного распределения**: равномерный расход бюджета в течение дня
4. **Заданной цели**: максимизация нужного показателя (клики, конверсии, показы)

### Виды автостратегий

| Стратегия | Когда использовать | Минимальные требования |
|-----------|-------------------|----------------------|
| Максимум кликов | Новый оффер, нужна массовость, сбор данных | Бюджет на 50+ кликов/день |
| Максимум конверсий | Есть цель в Метрике, стабильная форма | 10+ конверсий/неделю для обучения |
| Целевая CPA | Уверенная конверсионная воронка | 20+ конверсий/неделю |
| ROAS | Повторные продажи, CRM-данные | Передача revenue в Метрику |
| Пакетная | Объединение нескольких кампаний с общей целью | 2+ кампании с одной целью |

### Критические правила обучения

- **Grace period**: первые 10-14 дней -- ТОЛЬКО сбор данных, НЕ трогать
- **Смена стратегии = переобучение**: алгоритм откатывается к нулю
- **Смена креатива, лендинга или оффера = ломает обучение**: алгоритм теряет поведенческую историю
- **"Засыпание"**: автостратегия может перестать выкупать трафик -- нужен перезапуск
- **Нелинейность бюджета**: x2 бюджет != x2 конверсий по той же цене

### Чего НЕЛЬЗЯ делать во время обучения

1. Менять стратегию
2. Резко менять бюджет (более 20-30% за раз)
3. Менять цели в Метрике
4. Менять посадочную страницу
5. Вносить корректировки по аудиториям

## 2. Цели Метрики для оптимизации (Абрамова, урок 2)

### Типы целей и их влияние на автостратегию

Цели определяют, что алгоритм считает "успехом". Без целей автостратегия ориентируется на клики/показы -- это бессмысленно для бизнеса.

**Обязательные цели для юр. бизнеса:**
1. **Заявка с формы** (URL-страница: /thanks или JS-событие submit)
2. **Звонок** (через Calltracking -- коллтрекинг обязателен)
3. **Сообщение в мессенджер** (WhatsApp/Telegram -- JS-событие клика)

### Настройка целей в Метрике

1. Счётчик Метрики -> Цели -> Добавить цель
2. Типы: URL-страница, Событие (JS), JavaScript-событие
3. Каждой цели присвоить ценность (для стратегии "Максимум конверсий с оплатой за конверсии")
4. Связать счётчик с Директом

### Микроконверсии (легальные)

Для ускорения обучения автостратегии можно использовать промежуточные цели:
- **Скролл до формы** -- показывает интерес
- **Время на сайте > 30 сек** -- отсекает случайные клики
- **Просмотр ключевой страницы** (прайс, контакты) -- сигнал интента

**Правило**: промежуточная цель должна коррелировать с реальной конверсией. Нерелевантные цели "сгоняют" качество трафика.

### Ценности целей

В Директе можно присвоить разные ценности разным целям:
- Заявка = 5000 руб (примерная стоимость лида в банкротстве)
- Звонок = 5000 руб
- Мессенджер = 3000 руб (конверсия ниже)

Алгоритм оптимизирует под суммарную ценность, а не под количество.

## 3. Управление автостратегиями: тактики (Абрамова, урок 5)

### Слабости автостратегий (что использовать легально)

1. **На старте слепа** -- ей нужно "показать", на что ориентироваться. Решение: холодный старт через ручной режим (7-10 дней ручных ставок -> переключение на авто)
2. **Ориентирована на прошлое** -- инерция 7-14 дней после любого изменения
3. **Может "уснуть"** -- перестаёт выкупать трафик, если конверсий мало. Решение: перезапуск кампании

### Разделение воронки по стратегиям

Создать несколько РК с разными целями для разных этапов воронки:

| Кампания | Стратегия | Цель | Этап |
|----------|-----------|------|------|
| Поиск (холодный) | Макс. кликов | Трафик, тест заголовков | Верх воронки |
| Поиск (тёплый) | Макс. конверсий | Заявки, звонки | Середина воронки |
| Ретаргетинг | Макс. конверсий / CPA | Возврат и дожим | Низ воронки |

### Холодный старт через ручной режим

1. Запуск кампании с ручным управлением ставками
2. 7-10 дней сбора поведенческой истории
3. Переключение на автостратегию -- алгоритм получит "вводные" и обучится быстрее
4. **Grace period после переключения**: ещё 7-10 дней не трогать

### Когда перезапускать кампанию

- Конверсий 0 за 14+ дней при нормальном трафике
- CPA улетел в 3x+ от целевого и не снижается 7+ дней
- Алгоритм перестал тратить бюджет (показы стремятся к 0)
- После серьёзных изменений (смена лендинга, оффера)

### Пакетные стратегии

Объединяют несколько кампаний с общей целью. Полезно когда:
- У отдельных кампаний мало данных для обучения
- Нужно оптимизировать бюджет между регионами/сегментами
- Одинаковая цель, но разные ключи/аудитории

## 4. Региональные хаки (Абрамова, уроки 4, 7)

### Проблема: перегретые регионы

Юр. тематика (банкротство) -- перегретая ниша:
- Москва: CPC 140+ руб
- Казань/Набережные Челны: CPC 40-80 руб
- Малые регионы: CPC 15-37 руб

### Стратегия: разделение кампаний по регионам

**НЕ вести общую РК по всей России.** Создавать отдельные кампании:

1. **Набережные Челны + Казань** -- основной регион (услуги оказываются из 3 городов)
2. **Крупные города-миллионники** -- Екатеринбург, Самара, Уфа и т.д.
3. **Мелкие города** -- низкая конкуренция, дешёвый CPC
4. **Федеральная** -- вся РФ минус целевые регионы (для онлайн-банкротства)

**Преимущества разделения:**
- Тонкая настройка ставок под уровень конкуренции
- Разные УТП под регион
- Точный анализ рентабельности по регионам

### Практический кейс (Абрамова)

- Курс для водителей: Москва = 140 руб/клик
- Вся РФ минус Москва = 24-37 руб/клик + дополнительные лиды
- Экономия: в 4-6 раз на клике

### Адаптация УТП под регион (для банкротства)

| Регион | Акцент в УТП |
|--------|-------------|
| Москва/СПб | Премиум: "Ведущие юристы", срочность |
| Регионы-миллионники | Цена: "от 5900/мес", рассрочка |
| Малые города | Доступность: "Онлайн из любого города", бесплатная консультация |

### Шаблон {город} в заголовках

Яндекс подставляет город пользователя: "Банкротство в {город} от 5900/мес" -- повышает CTR на 15-30% за счёт персонализации.

### Важные правила

- Расширенный геотаргетинг ВЫКЛ для локального бизнеса
- Сравнивать регионы по стоимости ЦЕЛИ (CPA), не по CPC
- Отключать убыточные регионы еженедельно
- "Расползание" автостратегии за пределы заданного гео -- проверять в Метрике реальное гео

- [ ] РСЯ: ВКЛ только рекламная сеть, ВЫКЛ поиск
- [ ] Кампании разделены по регионам

### Настройки кампании

- [ ] Счётчик Метрики подключён
- [ ] Целевые действия настроены (заявка, звонок, мессенджер)
- [ ] UTM-метки на всех кампаниях
- [ ] Корректировки: < 18 лет = -100%, ТВ = -100%
- [ ] Расширенный геотаргетинг ВЫКЛ
- [ ] Мониторинг сайта ВКЛ
- [ ] "Директ помогает" ВЫКЛ
- [ ] ALTERNATIVE_TEXTS ВЫКЛ
- [ ] BidCeiling установлен
- [ ] Бюджет: минимум 15000-30000 руб/неделю на поиск

### Группы и ключи

- [ ] STAG: одна тема = одна группа
- [ ] Автотаргетинг на поиске: ТОЛЬКО "Целевые запросы"
- [ ] Автотаргетинг в РСЯ: можно все галочки
- [ ] Минус-слова: максимально подробные (221+ слов на кампанию)
- [ ] Ключи поиск: точные запросы с операторами
- [ ] Ключи РСЯ: широкие (тематика), минус-слова НЕ использовать

### Объявления

- [ ] Текстово-графические -- основной формат
- [ ] 2-3 варианта на группу (A/B тест)
- [ ] Конкретика в заголовках: цифры, проценты (не банальщина)
- [ ] Быстрые ссылки: минимум 3-4
- [ ] Уточнения: все заполнены
- [ ] Нет: CAPS, множественные !, смайлики, клише

### РСЯ

- [ ] 2 группы: по ключам + по интересам/конкурентам
- [ ] Карусель: 10 изображений
- [ ] Баннеры: объект по центру, мало текста
- [ ] Запрещены некачественные площадки (DSP, игры)

## 6. Принципы оптимизации (Идалин, 21 урок)

### Еженедельный цикл оптимизации (по понедельникам)

1. **CPA по кампаниям** -- найти аутсайдеров (CPA > 2x от среднего)
2. **Поисковые запросы** -- минусовать нецелевые
3. **Площадки РСЯ** -- запретить некачественные (DSP, игры, мусор)
4. **Пол/возраст** -- отключить неконвертирующие сегменты
5. **A/B тест объявлений** -- отключить слабые
6. **Бюджет** -- корректировка максимум +/-20-30%

### 4 стратегии Идалина

1. Максимум конверсий (оплата за клики или конверсии)
2. Максимум прибыли
3. Максимум кликов (с авто или ручными ставками)
4. Пакетная (объединяет несколько кампаний)

### Типовые проблемы и решения

| Проблема | Диагностика | Решение |
|----------|------------|---------|
| Высокие отказы в РСЯ | Отказы > 50% | Запретить площадки DSP и игры |
| Нецелевые запросы | Просмотр поисковых запросов | Еженедельная минусация |
| Рост CPA | CPA > 2x от целевого | Анализ по группам и пол/возрасту |
| Нет конверсий | 0 конверсий за 10+ дней | Проверить цели, лендинг, релевантность |
| Алгоритм "уснул" | Показы стремятся к 0 | Перезапуск кампании или смена цели |

### Бюджетные правила

- Увеличивать НЕ более чем на 20-30% в неделю
- Корректировки -- только по понедельникам
- Первые 10 дней -- только сбор данных
- Минимум 15000-30000 руб/неделю на поиск (юристы -- перегретая ниша)

### Юристы / банкротство -- специфика

- CPA 7000-40000 руб -- это НОРМА для ниши
- Обязательно: Calltracking + Метрика + UTM
- Сильные конкуренты уже всё оптимизировали -- на старте будет дорого
- Конверсия поиска: 3-8%, РСЯ: 0.5-2%

## 7. Decision Framework для AI-агента

### Когда ОСТАНАВЛИВАТЬ кампании

```
ЕСЛИ (расход > 6400 руб) И (конверсий == 0) И (дней > 7):
  → СТОП. Не тратить больше. Разобрать причины.

ЕСЛИ (отказы > 60%):
  → Проблема с лендингом или с нецелевым трафиком.
  → Проверить поисковые запросы и площадки.

ЕСЛИ (CPA > 40000 руб) И (кампания работает > 14 дней):
  → Стратегия нерентабельна. Сменить подход.

ЕСЛИ (показы упали > 80% от прошлой недели):
  → Алгоритм "уснул" или кончился бюджет.
  → Проверить баланс, перезапустить кампанию.
```

### Когда ДОБАВЛЯТЬ минус-слова

```
ЕЖЕНЕДЕЛЬНО (понедельник):
  1. Reports.get → отчёт по поисковым запросам за последние 7 дней
  2. Фильтр: запросы с кликами > 0 И конверсий == 0
  3. Ручная/AI проверка релевантности каждого запроса
  4. Нерелевантные → Keywords.delete или добавить в минус-слова кампании

АВТОМАТИЧЕСКИ:
  - CTR < 0.5% И клики > 10 И конверсий == 0 за 30 дней → удалить фразу
  - Запросы со словами "бесплатно", "скачать", "реферат", "курсовая" → минус
```

### Когда МЕНЯТЬ стратегию

```
НЕ МЕНЯТЬ если:
  - Прошло менее 14 дней с запуска/изменения
  - Есть конверсии в пределах целевого CPA
  - Тренд улучшается (CPA снижается неделя к неделе)

МЕНЯТЬ если:
  - 14+ дней, 0 конверсий при нормальном трафике (50+ кликов/неделю)
  - CPA стабильно > 3x от целевого после 21 дня
  - Алгоритм "уснул" (показы → 0)

ПОРЯДОК СМЕНЫ:
  1. Сначала попробовать сменить цель (на микроконверсию)
  2. Потом попробовать ручной режим на 7-10 дней → снова авто
  3. Последний вариант: перезапуск кампании с нуля
```

### Grace period после изменений

| Действие | Grace period | Что контролировать |
|----------|-------------|-------------------|
| Запуск новой кампании | 10-14 дней | Только расход и клики |
| Смена стратегии | 10-14 дней | Не менять ничего |
| Добавление минус-слов | 3-5 дней | Падение показов — норма |
| Смена бюджета (+/-20%) | 5-7 дней | CPA может скакнуть |
| Смена цели в Метрике | 14 дней | Полное переобучение |
| Корректировки по аудиториям | 5-7 дней | Объём трафика |

### Первые 10 дней -- протокол запуска

1. **День 0**: Запуск кампании. Записать начальные параметры (бюджет, стратегия, цели)
2. **День 1-3**: Только мониторинг. Проверять: модерация пройдена, объявления показываются, счётчик фиксирует визиты
3. **День 3-7**: Проверить поисковые запросы -- есть ли явный мусор. Если > 50% нецелевых → добавить минус-слова (единственное допустимое действие)
4. **День 7-10**: Первый срез: CPA, CTR, отказы. НЕ МЕНЯТЬ стратегию
5. **День 10-14**: Второй срез. Принятие решения: продолжать / корректировать / остановить

### Автоматический мониторинг (ежедневный cron)

```
08:00 — Reports.get за вчера:
  - Расход, клики, показы, CTR, CPA
  - Отправка сводки в Telegram

АЛЕРТЫ (отправить немедленно):
  - Расход за день > 2x от среднего дневного
  - 0 кликов за день (кампания "встала")
  - Отклонение объявлений модерацией
  - Баланс аккаунта < 3000 руб
```

### Еженедельная оптимизация (понедельник 09:00)

```
Шаг 1: Campaigns.get → статусы всех кампаний
Шаг 2: Reports.get → недельный отчёт по кампаниям
Шаг 3: Reports.get → поисковые запросы за неделю
Шаг 4: Анализ:
  - CPA по кампаниям → аутсайдеры?
  - Поисковые запросы → минусовать нецелевые
  - Площадки РСЯ → запретить мусор
Шаг 5: Keywords.update → корректировка ставок (если ручной режим)
Шаг 6: Keywords.delete → удалить мёртвые фразы
Шаг 7: Бюджет → +/-20-30% (если нужно)
Шаг 8: Отчёт с рекомендациями → Telegram
```

## 8. API: методы и лимиты

### Используемые методы

| Метод | Назначение | Частота |
|-------|-----------|---------|
| Campaigns.get | Статусы кампаний | 1 раз/день |
| Campaigns.suspend/resume | Остановка/запуск | По необходимости |
| AdGroups.get | Группы | 1 раз/неделю |
| Ads.get | Статусы объявлений | 1 раз/неделю |
| Keywords.get | Фразы со статистикой | 1 раз/неделю |
| Keywords.update | Корректировка ставок | 1 раз/неделю |
| Keywords.delete | Удаление мёртвых фраз | 1 раз/неделю |
| Reports.get | Статистика (TSV) | 1 раз/день |

### Лимиты

- Баллы (Units): если доступно < 100 → пауза до следующего часа
- Одновременные подключения: используем 1 из 12 доступных
- Размер: max 10000 объектов в get (Page: Limit 1000), max 1000 в add/update (батчи по 200)
- Отчёты: max 5 одновременных, мы запрашиваем 1
- Между запросами: минимум 1 секунда

### Обработка ошибок

| Код | Ошибка | Действие |
|-----|--------|---------|
| 52 | NoRights | СТОП, прекращаем |
| 53 | NotEnoughUnits | Пауза 1 час |
| 54 | ConcurrentLimit | Retry через 5 сек |
| 506 | EditingInProgress | Retry через 10 сек |
| 500 | Internal Error | Retry через 30 сек |
| 502 | Bad Gateway | Retry через 60 сек |
| 503 | Service Unavailable | Retry через 120 сек |

- **Конкуренция**: ВЫСОКАЯ, CPA 7000-40000 руб в нише
- **Регионы**: Набережные Челны, Казань, Нижнекамск + федеральные кампании
- **Ежемесячный бюджет**: 60 000-120 000 руб (пока не подтверждена рентабельность)

### Prelaunch checklist (обязателен перед каждым запуском)

1. Лендинг: отказы в Метрике < 40% на органике?
2. Формы: все работают, данные приходят в CRM?
3. Calltracking: подменные номера работают?

5. UTM-метки: стоят на всех ссылках?
6. Минус-слова: обновлены (221+)?
7. Бюджет: минимум 15 000 руб/неделю?
8. Расширенный геотаргетинг: ВЫКЛ?
9. "Директ помогает": ВЫКЛ?
10. ALTERNATIVE_TEXTS: ВЫКЛ?

## 11. Автостратегии: запуск и масштабирование (Цымбалист 2025)

**Источник:** Илья Цымбалист, fast-marketing.ru. Книга "Запуск и масштабирование рекламы в Яндекс.Директ на автостратегиях 2025". 69 страниц, 9 глав.

**Ключевая концепция:** Автотаргетинг и ключевые слова --- это один и тот же механизм. Разница лишь в исходных данных: ключи задаёте вы, а автотаргетинг берёт данные из заголовка и title сайта. Поэтому вопрос "что лучше" некорректен --- это один инструмент.

---

### 11.1. Фундаментальные правила

#### Разделение Поиска и РСЯ
- **НИКОГДА** не смешивать Поиск и РСЯ в одной кампании
- Причина: в РСЯ больше объём трафика, система сольёт туда весь бюджет, даже если Поиск эффективнее
- Разделение даёт: раздельное управление бюджетами, корректная аналитика, нет "перетягивания"

#### Режим создания кампании
- **Простой старт** --- нельзя, нулевой контроль
- **Мастер кампаний** --- нельзя, бюджет утекает в РСЯ, невозможно тестировать аудитории
- **Режим эксперта -> ЕПК** --- единственный правильный вариант

#### Распределение бюджета внутри кампании
- Яндекс НЕ тестирует группы поочерёдно --- выбирает 1-2 "перспективные" и вливает туда бюджет
- Перспективность = не только эффективность, но и размер аудитории (куда можно потратить больше)
- 80% денег в группе уходит на одно объявление, остальные практически не показываются

---

### 11.2. Настройка стратегии обучения

#### Рекомендуемая настройка
- Стратегия: **Максимум конверсий**
- С оплатой: **За клики** (НЕ за конверсии --- для новых аккаунтов не работает)
- Бюджет: **В неделю** (конкретная сумма)
- Ограничение расхода: **Бюджет** (НЕ "Средняя цена конверсии" --- лишает гибкости)

#### Когда использовать "Максимум кликов"
- Только при ретаргетинге на малую аудиторию (1000-2000 чел), где данных для оптимизации по конверсиям не хватит

#### Почему НЕ "Оплата за конверсии"
- Яндекс не знает сколько стоит ваша конверсия на новом аккаунте
- Если указать слишком низкую цену --- не найдёт аудиторию, не будет откручивать бюджет
- Работает только на давно работающих аккаунтах, когда вы готовы платить за конверсию в 1.5-2x дороже

---

### 11.3. Как автостратегия обучается

#### Ключевые пороги и сроки
- **Минимум 10 конверсий в неделю** на кампанию для обучения
- Результаты обучения **не обнуляются**: если на 1-й неделе 4 конверсии, на 2-й кампания уже работает лучше
- **НЕ использовать общий лимит на уровне аккаунта** --- будет принудительно останавливать кампании и сбивать обучение

#### Динамика обучения по дням
| День | Что происходит |
|---|---|
| 1-2 | Дорогие лиды (1700р при целевых 200р). Это нормально |
| 3-4 | Стоимость лида начинает снижаться |
| 5-7 | Кампания выходит на стабильные показатели, близкие к KPI |

**Критически важно:** Смотреть не на среднюю цифру за период, а на **положительную динамику по дням**. Если цена конверсии ежедневно падает --- всё идёт правильно.

#### Что портит обучение
- Отключение объявлений или групп внутри кампании в период обучения
- Чистка площадок РСЯ (бессмысленно --- Яндекс новых накидает, обучится сам)
- Изменение бюджета в пятницу-воскресенье
- Установка общего лимита на весь рекламный аккаунт

---

### 11.4. Правильный выбор цели (60% успеха)

#### Правило 1: Только конечная цель
- Цель = финальное действие на данном этапе воронки
- Пример: посещение страницы благодарности после заполнения формы, старт Telegram-бота
- **НЕ** покупку (если она через дни после вебинара --- атрибуция теряется)

#### Правило 2: НЕ цели на кликах
- Нельзя использовать "Клик по кнопке WhatsApp" или "Перейти в Telegram" как цель
- Система приведёт дешёвых кликеров, которые НЕ совершают следующее действие

#### Правило 3: НЕ созависимые цели в одной кампании
- Нельзя: регистрация + старт бота + посещение вебинара + покупка (шаги одной воронки)
- Яндекс воспримет как разные, несвязанные события --- обучение сломается
- Несколько целей можно только если они **независимы** (две разные формы заявки)

---

### 11.5. Работа с бюджетом

#### Особенности недельного бюджета
- Расход **неравномерный** по дням: может быть 2500р в один день и 300р в другой
- Яндекс обещает выдержать **общую сумму за календарную неделю** (пн-вс)
- При запуске в четверг с бюджетом 10К --- поставить сначала 5К, в понедельник увеличить до 10К

#### Правило выходных
- **НЕ запускать** новые кампании и НЕ менять бюджет в пятницу, субботу, воскресенье
- Причина: система попытается израсходовать весь недельный бюджет до конца воскресенья

#### Перерасчёт при изменении в середине недели
- Если в четверг увеличить бюджет с 5К до 7К, алгоритм постарается потратить **новые 7К с четверга по воскресенье**, проигнорировав уже потраченное

#### Работа с дорогими конверсиями (CPA 1200-2000р)
- Начать с 5000р/неделю
- Дождаться первых лидов, убедиться что связка работает
- Постепенно увеличивать
- **НЕ поддаваться соблазну** выбрать промежуточную цель ("посещение сайта 30 сек")
- Лучше необученная кампания на правильной цели, чем обученная на бесполезной

---

### 11.6. Типы аудиторий --- что работает

| Тип аудитории | Эффективность | Комментарий |
|---|---|---|
| **Автотаргетинг** | Высокая | Основной инструмент, лучше широких интересов и конкурентов |
| **Ключевые слова** | Высокая | Тот же механизм, только вы задаёте исходные данные |
| Ретаргетинг/LAL | Средняя | Хороши на старте, быстро "выгорают" |
| Готовые интересы Яндекса | Низкая | Дорогие "холодные" лиды, Яндекс вливает бюджет в бесполезную аудиторию |
| Сайты конкурентов | Низкая | НЕ аудитория конкурентов, а похожие. Редко лучше автотаргета |

#### Правила ключевых слов в РСЯ
- 2-3 слова во фразе (не длинные хвосты)
- НЕ больше 5-15 ключевых слов в группе
- Ключи работают "широкими мазками"

#### Тест ключи vs автотаргет
- ОТДЕЛЬНЫЕ кампании: одна с автотаргетингом, другая только с ключами
- В одной кампании --- автотаргет заберёт весь бюджет

---

### 11.7. Как автотаргетинг подбирает аудиторию (критически важно)

#### Что анализирует алгоритм
1. **80% внимания --- первый заголовок креатива** (второй заголовок, текст, быстрые ссылки --- почти не влияют)
2. **title страницы сайта** (не весь контент, а только title)

#### Практические примеры
- "Как говорить вкусно, уверенно и страстно" (ораторское) -> "вареники с руколой и шпинатом"
- "Реальные примеры миллионеров" (психология бизнеса) -> "как стать мега богатым"
- Одно слово "техника" вместо "как сделать" полностью меняет состав аудитории и конверсию
- "Баня на час" (лакшери загородный отдых) -> "баня на час в питере" (почасовая аренда)

#### Золотое правило заголовков
- **Контентное ядро:** точная фраза, описывающая продукт (= нацеливает автотаргетинг)
- **"Украшение":** маркетинговый крючок для клика (= не влияет на подбор аудитории)
- Пример: "Дрессировка собак [ядро] --- секреты от профессионалов [украшение]"

---

### 11.8. Структура кампаний и групп

#### Сколько групп в одной кампании
- **3-5 групп** --- золотая середина (протестируете несколько гипотез в рамках одного обучения)
- 10-15 групп: узнаете эффективность только 1 "победившей"
- 1 группа: нужен огромный бюджет на каждую аудиторию

#### Сколько объявлений в группе
- **Максимум 5** (больше --- система не успеет протестировать)
- Для начала достаточно 3
- НЕ делать десятки креативов, отличающихся только цветом фона

#### Принцип разделения гипотез
- Разные посылы = разные группы (НЕ в одной группе)
- Иначе алгоритм сольёт бюджет на самый широкий заголовок

#### Структура по бюджетам
- **15К/нед, лид 250р:** 2-3 кампании по 3-5К, 3-4 группы в каждой
- **10К/нед, лид 800р:** 1 кампания 5К, 2-3 аудитории. После первых лидов увеличить до 10К

---

### 11.9. Анализ эффективности

#### Поиск: CTR как индикатор
| CTR на Поиске | Оценка |
|---|---|
| 2-3% | Критически плохо. Автотаргетинг не понял аудиторию |
| 5-8% | Неудовлетворительно |
| **10-22%** | **Отлично, всё работает** |

Если CTR < 10% --- менять заголовки (контентное ядро).

#### Настройка отчёта "Поисковые запросы"
- Срезы: Поисковый запрос + Группы + Условие показа + Тип соответствия + Подобранная фраза + Категория таргетинга
- Столбцы: Показы, Клики, CTR, Расход, Ср. цена клика, Конверсия%, Цена цели, Конверсии
- Фильтр: **убрать** стандартный "Клики > 0", поставить "Показы > 0" --- видеть полную картину

#### РСЯ: Показатель отказов как индикатор
| Показатель отказов | Оценка |
|---|---|
| 15-20% | Норма (обучившаяся кампания) |
| 40-50% | Допустимо для широких ниш |
| 60%+ в первый день | **Нормально** для старта |

- 3-4 дня нет лидов, но отказы падают (60% -> 48% -> 31%) --- алгоритм обучается, ждать
- Динамика по дням важнее абсолютных цифр

#### Когда автотаргетинг НЕ работает
- Для узких ниш (газовое пожаротушение для серверных)
- Система цепляется за общие слова ("газ" -> "огнетушители", "газовое оборудование для дома")
- Решение: отключить автотаргетинг, работать ТОЛЬКО по ручным ключевым словам

---

### 11.10. Масштабирование бюджета

#### Как правильно увеличивать бюджет
- НЕ с 5К до 50К за один раз
- **Правильно:** 5К -> 9К (2-3 дня) -> 15К (2-3 дня) -> 25К
- После увеличения кампания может "тряхнуть" на 1-2 дня (рост CPA), затем стабилизируется
- **Все изменения --- с понедельника по четверг**

#### Почему бюджет может не откручиваться
1. Оцениваете по дням, а не по неделям
2. Установлено ограничение "средняя цена конверсии" (должно быть "бюджет")
3. Установлена "оплата за конверсии" (должна быть "за клики")

#### Рост CPA при масштабировании --- НОРМАЛЬНО
- Увеличение бюджета в 20 раз -> рост CPA в 3 раза
- Причина: для большего объёма система выкупает более дорогую аудиторию
- **Невозможно** масштабировать x10 и ожидать ту же цену лида

#### Создание новых кампаний при масштабировании
- НЕ плодить сразу (каждая проходит дорогой этап обучения)
- Создавать на основе работающих связок
- 5-10% бюджета на тест новых идей

#### Пример масштабирования
```
Старт: 2 кампании по 5К/нед (боль + тема)
-> Обучение -> Кампания 1 лучше -> 10К/нед
-> Кампания 1 приносит продажи -> 20К/нед
   Кампания 2 не приносит -> снизить до 7К
   Кампания 3 (новая, развитие лучшей гипотезы) -> 10К/нед
-> Кампания 1 -> 30К, Кампания 2 -> отключить, Кампания 3 -> 20К
-> Кампании 4, 5 --- тест новых гипотез по 5К
```

### 11.12. Decision rules (Цымбалист) для AI-агента

```
ЕСЛИ кампания < 7 дней И показатель отказов снижается по дням:
  -> ЖДАТЬ, не вмешиваться

ЕСЛИ CTR на Поиске < 10%:
  -> Менять заголовки (контентное ядро)

ЕСЛИ лиды дорогие первые 2-3 дня, но цена снижается:
  -> Нормальный процесс обучения, ЖДАТЬ

ЕСЛИ 7+ дней, < 10 конверсий, отказы не снижаются:
  -> Проверить: правильная ли цель? Не промежуточная?
  -> Проверить: не слишком ли узкая ниша для автотаргета?
  -> Проверить: заголовки точно описывают продукт?

ЕСЛИ кампания обучилась (10+ конверсий/нед, стабильная CPA):
  -> Масштабировать: текущий бюджет * 1.5-2x
  -> Ждать 2-3 дня стабилизации
  -> Создавать новые кампании на основе лучших связок

ЕСЛИ бюджет не откручивается после увеличения:
  -> Ограничение = "Бюджет" (не "Средняя цена конверсии")?
  -> Оплата за клики (не за конверсии)?
  -> Оцениваете по неделям, не по дням?

ЕСЛИ ниша слишком узкая (нецелевые запросы в отчёте):
  -> Отключить автотаргетинг
  -> Только ручные ключевые слова
```

### 11.13. Сравнение с другими источниками базы знаний

| Аспект | Идалин (секция 6) | Абрамова (секция 1-3) | Цымбалист (секция 11) |
|---|---|---|---|
| Автотаргетинг Поиск | Только "Целевые" | Не акцентировано | "Целевые" + "Узкие" |
| Бюджет: шаг увеличения | 20-30%/нед | 20-30%/нед | Более агрессивно: 5К->9К->15К->25К |
| Мониторинг сайта | ВКЛ | Не акцентировано | ВЫКЛ (может некорректно остановить) |
| Чистка площадок РСЯ | Запрещать DSP/игры | Запрещать мусор | Бессмысленно, обучится сама |
| Ограничение расхода | Не акцентировано | CPA / бюджет | Только "Бюджет", НЕ "Средняя цена конверсии" |
| Когда менять бюджет | По понедельникам | По понедельникам | Понедельник-четверг, НЕ пт-вс |
