# Минус-слова для БФЛ-кампаний (24bankrotstvo.ru)

Собрано из истории query_analyzer SDA v2.1 + доменная экспертиза подрядчиков.
Каждое слово/regex сопровождается причиной — агент цитирует её при
автоминусации.

## Категории

### 1. «Дешёвый / бесплатный» — люди без денег на услугу

**Regex:** `(?i)\b(дешев|бесплатн|скидк|акци[яию]|распродаж|со?\s*скидк)`

Причина: наши средний чек — 120-180К₽ за ведение БФЛ. «Бесплатно» = не наш сегмент. Конверсия <0.5%, сливает бюджет.

Примеры: «банкротство дешево», «бесплатная консультация банкротство», «банкротство со скидкой».

### 2. Отзывы / жалобы / разоблачения — информационный интент

**Regex:** `(?i)\b(отзыв|жалоб|разв[еo]д|лохотрон|мошенник|обман|кидал)`

Причина: пользователь ищет «развод ли банкротство» — ему нужна статья, не услуга. CR=0.

Примеры: «банкротство отзывы развод», «жалобы на банкротство», «банкротство мошенничество».

### 3. DIY — «сделать самостоятельно»

**Regex:** `(?i)(как\s+(сдел(а[тч]ь?)?|оформ|подат)\s+.*сам|самостоятельн|своими\s+силами|без\s+юрист)`

Причина: эти запросы = «инструкция, чтобы не платить». Наш продукт — сопровождение. Конверсия <1%, clogged форма заявок.

Примеры: «как подать на банкротство самостоятельно», «банкротство без юриста», «банкротство своими силами».

### 4. Юрлица — мы работаем только с физлицами

**Regex:** `(?i)\b(юр(идическ(ое|ого|их))?\s*лиц|ооо|зао|оао|пао|ип\b|индивидуальн(ых|ого)\s*предпринимат|компани[ияй]|организации?)\b`

Причина: БФЛ = Банкротство Физических Лиц. Банкротство компаний — другая процедура, другой ценник, мы не предоставляем.

Примеры: «банкротство ООО», «банкротство ИП», «банкротство юрлица», «банкротство компании».

### 5. Узкие услуги, которые мы не оказываем

**Regex:** `(?i)(налогов|алимент|по\s*кредит(у|ной)\s*карт|коллектор|приставов\s*жалоба|суд\s*с\s*банком)`

Причина: клиент ищет «как отменить алименты» — это не банкротство. «Коллекторы» — защита от коллекторов, отдельная услуга. Наши посадки сконвертят плохо.

Примеры: «банкротство по налогам», «банкротство по алиментам», «жалоба на приставов».

### 6. Образовательные / вакансии

**Regex:** `(?i)(курс|обучени[ея]|учебник|ваканси|работ(а|у|ать)\s+юристом|арбитражный\s+управляющий\s+вакан)`

Причина: это HR-запросы и курсы — не наш ICP.

Примеры: «курсы арбитражных управляющих», «вакансия юриста банкротство», «обучение банкротству».

### 7. Специфичные юрпроцедуры не под нами

**Regex:** `(?i)(упрощённ(ое|ый)\s*банкротст|через\s*мфц|мфц\s*банкрот|досудебн)`

Причина: **требует уточнения по бизнес-модели.** Внесудебное банкротство через МФЦ — бесплатно, мы его не сопровождаем за деньги. НО: на 24bankrotstvo.ru есть лендинг «через МФЦ» (консультационный) — возможно keep. Решение: **НЕ минусовать автоматом**, флагнуть для ручного решения.

Примеры: «банкротство через МФЦ», «внесудебное банкротство», «упрощённое банкротство».

### 8. Гео — если кампания не федеральная

**Regex:** Список названий других регионов, в которых мы НЕ ведём: «москва», «спб», «петербург», «екатеринбург», ...

Причина: 24bankrotstvo.ru — БФЛ в Приволжском ФО (Башкортостан/Татарстан/Удмуртия). Поиск из Москвы = лид невозможно сопровождать.

Mitigation: RegionIds на кампании уже ограничивают показ географически. Минус-слова гео = страховка от случайных показов в аудитории из других регионов (через VPN, командировки и т.п.).

## Неочевидные паттерны («выглядит целевым, но мусор»)

### Запрос упоминает сумму долга < 500К₽

**Regex:** `(?i)(долг\s*(до\s*)?[0-4]?\d{2}\s*(?:тыс|000))`

Причина: МРОТ на 2026 — 500К₽ минимум долга для БФЛ. При долгах <500К выгоднее реструктуризация/рефинанс, не банкротство. Иначе клиент разочаруется, churn.

Пример: «списать долг 300 тысяч».

### «Банкрот» как прилагательное (не процедура)

Например: «банкрот-тур», «банкрот-аукцион», «купить у банкрота квартиру».

Причина: человек хочет купить с торгов дешёвое имущество. Это B2B-сегмент для инвесторов, не должники.

**Regex:** `(?i)(банкрот[-\s]*(тур|аукцион|квартир|недвижим|имуществ|торг[иа]))`

## Как использовать в коде

`query_analyzer` job (Task 18) импортирует этот файл и парсит секции. Порядок применения:
1. Для каждого поискового запроса — прогнать по всем regex'ам.
2. Если match → rewrite как `-"<word>"` в минус-фразы ad-group.
3. Если в описании категории сказано «требует уточнения» (категория 7) — только NOTIFY, не AUTO.

Тесты: `test_minus_words_regex_coverage` в `tests/unit/test_query_analyzer.py` (Task 18) — проверяет что на sample queries (≥50 штук) F1 ≥ 0.85.
